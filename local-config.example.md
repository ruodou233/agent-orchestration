# agent-orchestration 本地配置示例

复制本文件结构到 `~/.config/agentops-skills/agent-orchestration/local-config.md`，或在只读安装场景下复制为 skill 目录内的 `local-config.md`。写入前应先向使用者说明写入位置并获得明确同意。

## 平台映射

| 平台 | 角色分工 | 工人默认侧 | 收敛位 | 档位描述 |
|---|---|---:|---:|---|
| 主平台 | 战略规划、用户沟通、指挥编排、最终决策 | 否 | 是 | 主代理按使用者当前会话设置；聚合、验收、评审使用当前可用强档 |
| 第二厂商 CLI | 批量调研、批量阅读、批量生成、可验证执行任务 | 是 | 否 | 执行工人默认强档；低风险、可重跑、低价值批处理可使用经济档 |
| 本地脚本/工具 | 统计、转换、格式化、schema 校验、批量文件处理 | 视任务而定 | 否 | 无模型档位；输出必须落盘，并由主平台或收敛位验收 |

## 编排默认值

| 配置项 | 示例值 | 说明 |
|---|---|---|
| single_platform_policy | 工人全放本侧 | 只有一个可用平台时，不做跨平台分流 |
| multi_platform_worker_policy | 执行工人放 token 充裕侧 | 多平台时，把自包含、可验证、重消耗任务放到更适合批量执行的一侧 |
| convergence_policy | 收敛位留主平台 | 聚合、验收、终审默认由主平台或当前可用强档完成 |
| default_fanout | 2-4 | 超过该范围时说明独立性和合并策略 |
| state_workspace | workspace/ | 每次长任务可在项目内新建独立工作区 |
| handoff_threshold | 接近上下文上限或阶段切换时 | 生成 handoff 并给出状态文件路径 |

## 只读探测记录

| 探测项 | 结果示例 | 备注 |
|---|---|---|
| 已装 Agent CLI | 主平台 CLI、第二厂商 CLI | 只记录抽象名称即可 |
| 支持并行子代理 | 主平台：支持；第二厂商 CLI：支持独立会话 | 以实际平台能力为准 |
| 订阅或额度约束 | 主平台适合收敛；第二厂商 CLI 适合批量执行 | 不写具体付费或额度信息 |
| 可写配置位置 | 用户确认后写入本文件 | 未确认前不得写配置 |

## 任务级覆盖项

| 配置项 | 示例值 | 说明 |
|---|---|---|
| max_parallel_workers | 3 | 单次任务最大并发工人数 |
| max_retries_per_method | 2 | 同一方法失败达到该次数后换方法或降级 |
| require_independent_review | true | 交付前是否强制独立验收 |
| external_side_effects_allowed | false | 默认不允许外部副作用，除非使用者明确授权 |
